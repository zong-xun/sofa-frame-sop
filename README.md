# 沙發木架 SOP 總看板（靜態展示版）

給工廠同仁查看各沙發型號組裝步驟、找材料放置位置、看圖例規範說明的網站。純 HTML + CSS + JS，沒有後端、沒有資料庫，可以直接部署到 GitHub Pages。整個網站是單頁應用（點分頁/型號不會整頁重新整理），畫面固定滿版高度，「上一步／下一步」按鈕永遠在畫面下方，不用捲動就點得到。

## 三個分頁

- **📋 型號 SOP**：型號列表 → 點進去看該型號的組裝步驟，一步一步的做法、注意事項、示意圖，可用「上一步／下一步」按鈕、點步驟圓點、鍵盤左右鍵、手機滑動切換
- **🔍 找料位置**：倉庫分區地圖，點格子或右側材料清單，看某個代號的材料規格、用在哪些型號、放在哪一區
- **📐 圖例規範**：說明步驟示意圖上紅圈（釘點）、藍圈（對齊基準）、箭頭（施工方向）、數字圈（組裝順序）分別代表什麼

## 本機預覽

用 VSCode 的 "Live Server" 套件，右鍵 `index.html` → "Open with Live Server"，或直接雙擊 `index.html` 用瀏覽器開啟。

## 目前的示範內容

放了三組示範型號（A-01 三人座、B-02 二人座、L-03 轉角），步驟圖是用程式碼畫的示意圖（不是真實照片），先感受操作方式，之後再換成真實照片即可。

## 如何修改內容

所有內容都在 [js/data.js](js/data.js) 這支檔案，其他程式檔案不用動。

### 修改／新增型號的 SOP 步驟

`MODELS` 陣列裡每個型號一個物件，`steps` 是步驟清單，每個步驟：

```js
{
  name: "釘槍釘點—椅背",           // 步驟名稱
  desc: "沿椅背框邊緣打釘...",      // 做法說明
  warn: "轉角處記得加補一釘。",      // 注意事項（沒有就寫 ""）
  img: "",                        // 有真實照片時，填路徑例如 "images/A-01/step3.jpg"
  art: frame(3),                  // 沒填 img 時顯示的自動示意圖，數字 1~5 對應五種常見階段（見下方）
}
```

`frame(數字)` 是自動畫示意圖用的，對應五種階段：`1`檢料、`2`組裝底框、`3`中間補強、`4`側柱／扶手、`5`驗收。步驟不到 5 步就跳著用（例如只有 4 步，就用 `frame(1) frame(2) frame(3) frame(5)`）。

### 新增真實照片

1. 在 `images/` 資料夾建一個以型號代號命名的子資料夾（例如 `images/a-01/`），把照片放進去
2. 該步驟的 `img` 欄位填照片路徑，網頁就會改顯示照片，不會再顯示自動示意圖

### 修改找料位置的內容

`ZONES`：倉庫分區（代號、名稱、顏色、底下有哪些格子代號）
`MATERIALS`：每個格子代號實際放什麼料（代號、名稱、規格、用在哪些型號代號）

### 修改圖例規範說明

`RULES` 陣列，每一條是一個符號的說明（圖示、標題、說明文字）。

## 部署到 GitHub Pages

1. 把這個資料夾整個 push 到一個 GitHub repository
2. 進到 repo 的 **Settings → Pages**
3. Source 選擇 `Deploy from a branch`，Branch 選 `main`、資料夾選 `/ (root)`
4. 存檔後等 1-2 分鐘，GitHub 會給一個網址（例如 `https://你的帳號.github.io/repo名稱/`），之後同仁用手機或工廠電腦瀏覽器打開這個網址就能看到

之後每次修改 `js/data.js` 或新增照片，git push 上去，GitHub Pages 會自動更新。

## 之後可以考慮的升級方向（目前不做）

- 後台管理介面：讓爸爸不用改程式碼、直接在網頁上新增型號/步驟/上傳照片
- 帳號登入保護：避免內容被外部搜尋到或被修改
- 資料庫：型號數量變多、需要多人同時維護時再考慮
